關於 Clipper

Clipper 是一套內網跨子網 P2P 多人協作工具，透過 WebRTC 建立點對點連線， 搭配 WebSocket 信令伺服器進行配對與持久化資料儲存。支援多達 50 台裝置同時連線。 採用模組化架構，各功能獨立隔離，單一模組故障不影響其他功能。

版本v2.0.0
作者Hugo Leung
技術WebRTC · WebSocket · Python · SQLite · Vanilla JS
前端架構7 ClipperModule — 故障隔離
後端架構7 Service 模組 · 路由表
授權MIT

🚀 功能

  • 💬 實時短信 — P2P 即時訊息，使用者頭像色碼 + 漸層泡泡
  • 📁 檔案傳輸 — 拖放上傳，佇列傳送，SHA-256 完整性驗證
  • 📋 公告欄 — 4 級分類、標籤、釘選、側邊欄一覽
  • ✅ 檢查清單 — 階層式 Board、10 色色盤、釘選、排程提醒、JSON/Markdown 匯出入
  • 🔑 密鑰管理 — 串流金鑰 CRUD，啟用/停用切換
  • 📚 範本庫 — 檢查清單範本建立/套用
  • 🧪 系統診斷 — 即時健康儀表板、Event Loop 監控、Sparkline 走勢、事件時間線
  • ⚙️ 設定頁 — P2P / Relay 傳輸模式檢視、NTP 時間同步
  • 🖥️ 伺服器持久化 — SQLite 統一儲存（自動備份、匯入匯出）
  • 🔍 錯誤聚合 — 所有例外集中記錄，可視化查閱
  • 🛡️ 離線唯讀防護 — 斷線自動鎖定，重連自動恢復
  • 🧟 幽靈復活防護 — 刪除 ID 伺服器記錄，重連不過濾
  • 👥 Full Mesh 網狀網路 — 最多 50 台同時連線
  • 🏷️ 顯示名稱系統 — 隨機中文暱稱，跨 F5 保留
  • 🕐 頁頂時鐘 — 即時 hh:mm:ss 及當日日期
  • 📱 自適應佈局 — 響應式斷點 768px，通知側欄可切換

🔒 資訊安全提醒

⚠️ 本應用僅供內網使用，請注意以下事項：

  • 所有 P2P 訊息經 DTLS 加密，但 不建議 在不受信任的網路上傳輸敏感資料
  • 信令伺服器本身以明文 WebSocket 運行（ws://），配對及 Relay 過程無加密
  • 擁有配對代碼的任何用戶均可加入房間並存取所有共享資料
  • 聊天訊息會備份至伺服器（留存天數可調），請勿傳送機密對話
  • 建議在公司內部可信任網路中使用，若需跨網際網路請自行配置 WSS/TLS

⚠️ 已知限制

  • 需要 Python 3.8+ 執行信令伺服器（可 Docker 化或打包為二進位檔）
  • 跨子網需要網路設備支援 UDP 路由 — 已內建 Google STUN 伺服器（設定頁可修改）， 若 STUN 仍無法穿透則自動降為 WS Relay 中繼
  • 檔案傳輸支援多對象並行傳送（各對象獨立不阻塞），P2P 失敗時自動降為 WS Relay
  • 房間上限 50 人（可調整 MAX_PEERS_PER_ROOM），建議 20 人以內以保持 P2P 效能
  • 標準 TURN 協議未內建（因為 WS Relay 已提供同等中繼功能），設定頁可自訂 STUN/TURN 位址
