關於 Clipper

Clipper 是一套內網跨子網 P2P 多人協作工具，透過 WebRTC 建立點對點連線， 搭配 WebSocket 信令伺服器進行配對與持久化資料儲存。支援多達 50 台裝置同時連線。

版本v1.1.0
作者Hugo Leung
技術WebRTC · WebSocket · Python · WebSockets
授權MIT

🚀 功能

  • 💬 實時短信 — 即時 P2P 訊息廣播，自動 WS Relay 後備
  • 📁 檔案傳輸 — 拖放上傳，支援大檔案區塊傳送，指定對象傳送
  • 📋 公告欄 — 分類（重要/日常/交接事項/其他事項）、標籤、色碼、釘選
  • ✅ 檢查清單 — 階層式 Board、10 色色盤、標籤、釘選、排程提醒、Tick Reset
  • ⚙️ 設定頁 — 檢視傳輸模式（P2P / Relay）及各用戶連線狀態
  • 🖥️ 伺服器持久化 — 所有資料儲存於 vcc_server_state.json
  • 🔍 Debug Dump — 一鍵輸出伺服器完整診斷
  • 👥 Full Mesh 網狀網路 — 最多 50 台同時連線
  • 🏷️ 顯示名稱系統 — 隨機中文暱稱，跨 F5 保留
  • 🕐 頁頂時鐘 — 即時 hh:mm:ss 及當日日期
  • 🔗 預設配對碼 — 開啟頁面即自動連接 room 1234
  • 🔔 排程提醒 — 檢查清單可設定時間，到期自動彈窗通知
  • 🔄 手動同步 — 狀態指示器 + 同步按鈕
  • 📱 自適應佈局 — 視窗縮小時排版自動折疊

⚠️ 已知限制

  • 需要 Python 3.8+ 執行信令伺服器（可 Docker 化或打包為二進位檔）
  • 跨子網需要網路設備支援 UDP 路由 — 已內建 Google STUN 伺服器（設定頁可修改）， 若 STUN 仍無法穿透則自動降為 WS Relay 中繼
  • 檔案傳輸支援多對象並行傳送（各對象獨立不阻塞），P2P 失敗時自動降為 WS Relay
  • 房間上限 50 人（可調整 MAX_PEERS_PER_ROOM），建議 20 人以內以保持 P2P 效能
  • 標準 TURN 協議未內建（因為 WS Relay 已提供同等中繼功能），設定頁可自訂 STUN/TURN 位址
