# Clipper — 內網跨子網 P2P 多人協作工具

v1.1.0 — 支援公告欄、檢查清單、伺服器持久化、WS Relay 後備

## 快速開始

1. **安裝依賴：**
   ```bash
   pip install websockets
   ```

2. **啟動信令伺服器：**
   ```bash
   python3 signal_server.py
   ```
   伺服器會自動載入先前儲存的資料，啟動後監聽 `ws://localhost:8765`。

3. **開啟用戶端：**
   在瀏覽器（或多個分頁/視窗）中開啟 `clipper.html`。

4. **自動連線：**
   頁面載入後會自動使用預設配對碼 **1234** 建立房間。
   也可自行輸入其他代碼後點「加入配對」。

5. **開始使用！**
   所有功能在單人模式下亦可使用（資料儲存於伺服器）。

## 🚀 功能

### 💬 實時短信
- 即時 P2P 訊息廣播，自動顏色區分不同用戶
- 訊息持久化 — F5 重整不消失（localStorage 最多 200 則）
- 伺服器備份 — 可設定保留天數（預設 7 天）
- 「🗑 清除本機紀錄」按鈕（含確認對話框）
- WebRTC DC 失敗時自動降為 WS Relay 中繼

### 📁 檔案傳輸
- 拖放上傳，支援大檔案區塊傳輸與進度條
- 指定對象傳送（需手動選取傳送對象）
- 錯誤狀態顯示與重試功能

### 📋 公告欄
- 完整的公告 CRUD（新增 / 編輯 / 刪除 / 釘選）
- 4 級分類：**重要🔴** · **日常🔵** · **交接事項🟡** · **其他事項⚪**
- 標籤 badges、自動色碼
- 「重要」公告刪除有額外安全確認
- 經 WebSocket 同步至所有用戶

### ✅ 檢查清單
- 階層式 Board 結構，每個 Board 可獨立命名、設色、加標籤
- 10 色色盤（天藍/綠/紅/琥珀/紫/粉/青/橙/灰/白）
- 待辦項目新增 / 勾選 / 刪除
- **🔄 Tick Reset** — 一鍵重設所有勾選
- **🔔 排程提醒** — 設定日期時間，到期自動彈窗通知
- Board 級釘選、可收合
- 經 WebSocket 同步至所有用戶

### ⚙️ 設定頁
- 即時檢視傳輸模式（🟢 P2P / 🟡 WS Relay）
- 各用戶連線狀態列表
- 最後同步時間、房間代碼

### 🖥️ 伺服器 (signal_server.py)
- 公告欄、檢查清單、聊天備份全資料持久化（JSON 檔案）
- `vcc_server_state.json` 自動存檔／載入
- 可設定聊天備份留存天數（`CHAT_RETENTION_DAYS`）
- **WS Relay** — WebRTC 失敗時自動轉為伺服器中繼
- **Debug Dump** — 用戶端一鍵輸出完整診斷資訊
- Verbose DEBUG 模式（`DEBUG = True`）

### 👥 網路
- Full Mesh WebRTC 網狀架構，最多 50 台同時連線
- WebSocket 中繼後備方案（WebRTC 失敗時自動切換）
- 新成員加入自動與所有人配對
- 斷線自動清理，可重新連線

### 🏷️ 顯示名稱系統
- 隨機中文暱稱（形容詞 + 動物 + 數字，如「快樂熊貓42」）
- 可點擊編輯自訂名稱
- 跨 F5 保留（localStorage `vcc_display_name`）

### 🔍 其他功能
- 🕐 頁頂即時時鐘（hh:mm:ss + 當日日期）
- 🔗 預設配對碼 1234，開啟頁面即自動連線
- 🟢 同步狀態指示器 + 手動同步按鈕
- 📱 自適應佈局（視窗縮小時排版自動折疊）
- 🔤 全介面字體放大 25% 提升可讀性
- 🔔 Popup 通知 — 新公告/新檢查清單時右下角彈窗
- 所有刪除動作皆有確認對話框防誤刪
- Room-State Merge — 同步時合併而非覆寫本機資料

## 專案架構

```
├── clipper.html      # 主用戶端 SPA（WebRTC + WS + 所有功能模組，~2,900 行）
├── signal_server.py      # 信令伺服器（WebSocket 配對 + 持久化儲存 + Relay，~630 行）
├── CHANGELOG.md          # 版本變更記錄
├── README.md             # 本文件
└── vcc_server_state.json # 伺服器自動產生的持久化資料（公告欄、檢查清單、聊天備份）
```

### 通訊協議

```
WebSocket (WS)  ── 信令配對 + 公告/清單 CRUD + 聊天備份 + Debug Dump + Relay 中繼
DataChannel (DC) ── 實時短信 P2P + 檔案傳輸（即時通訊，最優先，失敗自動降為 WS Relay）
WS Relay         ── 當 DC 無法建立時，自動降級為 WebSocket 中繼傳輸
```

## 手動 E2E 測試流程

### 前置準備

1. 啟動信令伺服器：`python3 signal_server.py`
2. 確認終端機顯示 `listening on ws://localhost:8765`
3. 在多個瀏覽器分頁中開啟 `clipper.html`（或 `http://localhost:8765` 透過 HTTP 提供）

### 測試步驟

| 步驟 | 動作 | 預期結果 |
|------|------|----------|
| **1. 配對連線** | 開啟兩個分頁，自動使用 1234 配對 | 兩端狀態顯示「🟢 已連線」 |
| **1b. 自動同步** | 在分頁 A 新增公告 | 分頁 B 自動出現該公告 |
| **2. 實時短信** | 分頁 A 輸入文字後按 Enter | 分頁 B 收到訊息，顯示不同顏色 |
| **2b. 顯示名稱** | 點擊頂部的顯示名稱 → 編輯新名稱 → 按 Enter | 下一則訊息以新名稱顯示，F5 後仍保留 |
| **3. 檔案傳送** | 分頁 A 選取對象後拖放檔案到傳輸區 | 分頁 B 自動下載該檔案 |
| **4. 公告欄** | 分頁 A 新增公告 + 選擇分類 + 輸入標籤 | 分頁 B 顯示分類色碼與標籤 badges |
| **5. 檢查清單** | 分頁 A 新增 Board + 新增待辦項目 | 分頁 B 顯示 Board 與項目 |
| **5b. 排程提醒** | 設定提醒時間 | 時間到時雙方彈窗通知 |
| **5c. Tick Reset** | 點 🔄 按鈕 | 所有勾選重設為未完成 |
| **6. 同步狀態** | 檢視 footer 同步指示器 | 顯示 🟢 剛剛同步 |
| **7. Debug Dump** | 在關於頁點 Debug Dump | 主控台輸出完整伺服器診斷 |
| **8. 中斷連線** | 關閉伺服器後重新啟動 | 客戶端顯示「🔴 未同步」，重連後自動合併 |
| **9. 傳輸模式** | 檢視設定頁 | 顯示 🟢 P2P 或 🟡 Relay 模式 |

## 設定

### 伺服器 (signal_server.py)

```python
CHAT_RETENTION_DAYS = 7    # 聊天備份留存天數（可調整）
DATA_FILE = "vcc_server_state.json"  # 持久化檔案路徑
DEBUG = True               # Verbose 除錯日誌（False 只顯示主要事件）
```

### 用戶端 (clipper.html)

- 配對碼預設爲 `1234`，可在 footer 輸入框自訂
- 伺服器 URL 預設 `ws://localhost:8765`

## 已知限制

### 已解決 ✅

| 限制 | 解決方案 |
|------|---------|
| 跨子網 UDP 穿透 | 內建 Google STUN 伺服器（`stun:stun.l.google.com:19302`，設定頁可修改），若 P2P 仍失敗自動降為 WS Relay |
| 多人同時傳檔阻塞 | 各對象獨立並行傳送佇列，A 傳 B 不影響 C 傳 D |
| 檔案傳輸僅支援 P2P | 已實作 WS Relay（relay-chunk），P2P 失敗時自動以 base64 經伺服器中繼 |
| 無 STUN/TURN | 內建 Google STUN + 設定頁自訂；WS Relay 作為 TURN 替代方案 |

### 仍待解決

| 限制 | 建議解法 |
|------|---------|
| 需要 Python 3.8+ | 使用 Docker：`docker run ...` 或 PyInstaller 打包成單一執行檔 |
| 房間上限 50 人 | `MAX_PEERS_PER_ROOM` 可調整，建議 20 人以內維持 P2P 效能 |
| 大檔案 WS Relay overhead | 小檔案 (<10MB) 直接走 Relay；大檔案建議確保 STUN 穿透以使用 P2P（無 base64 開銷）|
| 無標準 TURN 協議 | WS Relay 已提供對等中繼功能；如有需要可在設定頁填入外部 TURN 伺服器 |
